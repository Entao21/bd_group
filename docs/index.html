Project Summary

This project develops an interactive Google Earth Engine application to explore glacier retreat susceptibility across Greenland’s peripheral glacier zone. The app combines Landsat-derived glacier change, Landsat land surface temperature trends, ArcticDEM terrain variables, and proximity indicators for mining activity, settlements, and coastlines. A Random Forest model is trained on observed retreat and stable-ice labels to map where current glaciers are more susceptible to retreat.

The purpose is not to prove that mining caused glacier retreat. Instead, the application provides a screening tool for identifying places where observed retreat, climate warming, terrain conditions, and mining-related proximity overlap. This supports more transparent discussion around Arctic resource development, environmental monitoring, and glacier vulnerability.

Problem Statement

Greenland’s peripheral glaciers are changing rapidly under Arctic warming, while renewed mineral exploration creates planning questions in sensitive cryospheric environments. Existing glacier products and mining datasets are usually viewed separately, making it difficult for non-specialist users to compare climate-driven change with human activity indicators.

This project addresses that gap by building a web-based decision-support tool that:

  • maps glacier extent change from approximately 2000 to 2024;
  • estimates summer land surface temperature trends;
  • integrates elevation, slope, mining proximity, settlement proximity, and coastal proximity;
  • predicts retreat susceptibility using a validated Random Forest model;
  • communicates model outputs through an interactive split-map interface.

End User

The application is designed for users who need a clear spatial overview rather than a static scientific map. Potential users include Arctic environmental researchers, Greenland planning and resource governance agencies, environmental NGOs, mining impact analysts, and students studying remote sensing or climate risk.

The interface supports these users by combining technical model outputs with accessible interaction: users can compare historical and recent glacier extent, toggle model layers, inspect model diagnostics, and click anywhere on the map to retrieve local glacier, temperature, elevation, and susceptibility statistics.

Methodology

The core analytical challenge is translating decades of raw satellite observations into a spatially consistent, interpretable susceptibility signal across Greenland’s peripheral glacier zone. Rather than relying on a single data source or a pre-classified product, this project combines multi-sensor remote sensing, per-pixel trend estimation, and machine learning to build a predictor stack that captures climate, terrain, and human-environment interactions simultaneously. Each methodological choice — from the NDSI threshold to the spatial block validation scheme — is designed to minimise label noise and reduce the risk of overfitting to spatially autocorrelated observations, so that the final susceptibility surface reflects genuine environmental gradients rather than artefacts of the training sample.

Glacier extent was mapped from cloud-masked Landsat summer composites (June–August) for two epochs: ~2000 (1999–2002) and ~2024 (2022–2024). The Normalized Difference Snow Index (NDSI) was computed from the green and SWIR bands of each sensor — Bands 2 & 5 for Landsat 7; Bands 3 & 6 for Landsat 8/9 — and thresholded at > 0.4 to classify ice-covered pixels. Retreat was identified as any pixel glaciated in ~2000 but bare in ~2024. To remove classification noise from cloud remnants, seasonal snow, and Landsat 7 SLC-off striping, connected-component filtering was applied to exclude isolated patches smaller than three contiguous pixels from both retreat and stable-ice labels. Terrain variables (elevation, slope) were derived from ArcticDEM v4 and all layers were standardized to a 300 m polar grid (EPSG:3413).

// Cloud masking using QA_PIXEL for Landsat 8/9
function maskL8(image) {
  var qa = image.select('QA_PIXEL');
  return image.updateMask(
    qa.bitwiseAnd(1 << 3).eq(0)   // cloud
      .and(qa.bitwiseAnd(1 << 4).eq(0))  // cloud shadow
  );
}

// NDSI from green and SWIR bands (Landsat 8/9)
function ndsiL8(img) {
  return img.addBands(
    img.normalizedDifference(['SR_B3', 'SR_B6']).rename('NDSI'));
}

// Build median NDSI composite and threshold at 0.4
function getGlacierExtent(startYear, endYear) {
  var t0 = ee.Date.fromYMD(startYear, 1, 1);
  var t1 = ee.Date.fromYMD(endYear, 12, 31);
  var summer = ee.Filter.calendarRange(6, 8, 'month');

  var l8 = ee.ImageCollection('LANDSAT/LC08/C02/T1_L2')
    .filterBounds(greenlandBox).filterDate(t0, t1)
    .filter(summer).map(maskL8).map(ndsiL8);
  var l9 = ee.ImageCollection('LANDSAT/LC09/C02/T1_L2')
    .filterBounds(greenlandBox).filterDate(t0, t1)
    .filter(summer).map(maskL8).map(ndsiL8);

  return l8.merge(l9).select('NDSI').median()
    .gt(0.4).rename('ice').clip(AOI_geom).unmask(0);
}

var glacier_2000 = getGlacierExtent(1999, 2002).rename('glacier_2000');
var glacier_2024 = getGlacierExtent(2022, 2024).rename('glacier_2024');

// Retreat = ice in 2000 but not in 2024
var retreatCandidate = glacier_2000.subtract(glacier_2024).max(0).gt(0);
var stableCandidate  = glacier_2000.eq(1).and(glacier_2024.eq(1));

// Remove isolated patches smaller than minPatchPixels (set to 3)
var cleanRetreat = retreatCandidate.selfMask()
  .connectedPixelCount(8, true).gte(minPatchPixels)
  .unmask(0).toByte().rename('retreat');

var cleanStable = stableCandidate.selfMask()
  .connectedPixelCount(8, true).gte(minPatchPixels)
  .unmask(0).toByte().rename('stable');

Summer land surface temperature (LST) trends were estimated from Landsat ST bands spanning 24 boreal summers (2001–2024). For each year, cloud-screened June–August scenes from Landsat 7, 8, and 9 were converted from raw digital numbers to degrees Celsius (scale factor 0.00341802; offset 149.0 − 273.15 K), then averaged into an annual composite. A per-pixel ordinary least squares linear regression across all 24 annual means produced a warming rate map in °C per year. Snapshot composites for ~2000 and ~2024 were additionally retained as absolute LST layers for direct visual comparison in the split-panel interface. These climate variables, together with terrain and anthropogenic proximity predictors, form the seven-variable stack used by the susceptibility model:

// Convert Landsat ST_B10 DN to Celsius
function lstL8(image) {
  return image.select('ST_B10')
    .multiply(0.00341802).add(149.0).subtract(273.15)
    .rename('LST_C')
    .copyProperties(image, ['system:time_start']);
}

// Build annual summer LST composites for 2001-2024
var years = ee.List.sequence(2001, 2024);

var annualLST = ee.ImageCollection(years.map(function(year) {
  year = ee.Number(year);
  var s = ee.Date.fromYMD(year, 6, 1);
  var e = ee.Date.fromYMD(year, 8, 31);

  var l8 = ee.ImageCollection('LANDSAT/LC08/C02/T1_L2')
    .filterBounds(greenlandBox).filterDate(s, e)
    .map(maskL8).map(lstL8);
  var l9 = ee.ImageCollection('LANDSAT/LC09/C02/T1_L2')
    .filterBounds(greenlandBox).filterDate(s, e)
    .map(maskL8).map(lstL8);

  return l8.merge(l9).mean()
    .set('year', year)
    .set('system:time_start', s.millis());
}));

// Attach numeric time band for OLS regression
var withTime = annualLST.map(function(img) {
  var yr = ee.Number(img.get('year'));
  return img.addBands(ee.Image.constant(yr).float().rename('time'));
});

// Per-pixel OLS slope = warming rate in °C/yr
var warmingRate = withTime.select(['time', 'LST_C'])
  .reduce(ee.Reducer.linearFit())
  .select('scale')
  .clip(AOI_geom)
  .rename('warming_rate');

// Proximity rasters: inverse cumulative cost distance, scaled 0-1
function distanceToFC(fc, maxDist) {
  var source = ee.Image(0).byte().paint(fc.map(function(f) {
    return f.buffer(1000);
  }), 1);
  return ee.Image(1).cumulativeCost(source, maxDist).unmask(maxDist);
}

var mineProximity = ee.Image(1)
  .subtract(distanceToFC(allMines, 100000).divide(100000))
  .max(0).min(1).rename('mine_proximity');

var settlementProximity = ee.Image(1)
  .subtract(distanceToFC(allSettlements, 100000).divide(100000))
  .max(0).min(1).rename('settlement_proximity');

var coastProximity = ee.Image(1)
  .subtract(distToCoast.divide(300000))
  .max(0).min(1).rename('coast_proximity');

A Random Forest regression model (100 trees) was trained to predict the susceptibility of present-day glaciers to retreat using the seven-variable predictor stack. The model domain was restricted to GIMP-identified land ice below 2500 m elevation, with an explicit inclusion of all ice south of 64°N to capture the southern Greenland ice cap, which behaves as peripheral glacier rather than interior ice sheet. Training labels were drawn exclusively from cleaned retreat and stable-ice pixels, excluding ambiguous intermediate areas.

To reduce spatial autocorrelation leakage between training and test samples, spatial block cross-validation was applied: the bounding box was tessellated into 10 geographic folds, with folds 0–6 (70 %) used for training and folds 7–9 (30 %) as the held-out test set. A separate classification-mode Random Forest was run on the same split for performance evaluation (accuracy, kappa, precision, recall, F1). The final susceptibility surface (0–1) represents the proportion of decision trees voting for retreat, applied exclusively to current (2022–2024) glacier pixels.

// Spatial block folds: divide lon/lat grid into 10 folds
var lonLat = ee.Image.pixelLonLat();
var spatialFold = lonLat.select('longitude').add(180).divide(2).floor()
  .add(lonLat.select('latitude').add(90).divide(2).floor().multiply(13))
  .mod(10)
  .rename('spatial_fold');

// Only sample from cleaned retreat and stable pixels within model domain
var sampleMask = cleanRetreat.eq(1).or(cleanStable.eq(1)).and(peripheralMask);

var trainingImage = predictors.addBands(target).addBands(spatialFold)
  .updateMask(sampleMask);

var samples = trainingImage.stratifiedSample({
  numPoints: 2000,
  classBand: 'retreat',
  region: greenlandBox,
  scale: exportScale,
  seed: 42,
  geometries: false,
  tileScale: 4
});

// 70% train (folds 0-6), 30% test (folds 7-9)
var trainSet = samples.filter(ee.Filter.lt('spatial_fold', 7));
var testSet  = samples.filter(ee.Filter.gte('spatial_fold', 7));

// Classification RF for validation metrics
var rfClassify = ee.Classifier.smileRandomForest({
  numberOfTrees: 100, seed: 42
}).train({
  features: trainSet,
  classProperty: 'retreat',
  inputProperties: bandNames
});

// Regression RF for continuous susceptibility score
var rfPredict = ee.Classifier.smileRandomForest({
  numberOfTrees: 100, seed: 42
}).setOutputMode('REGRESSION').train({
  features: trainSet,
  classProperty: 'retreat',
  inputProperties: bandNames
});

// Evaluate on held-out spatial blocks
var testClassified  = testSet.classify(rfClassify);
var confusionMatrix = testClassified.errorMatrix('retreat', 'classification', [0, 1]);
var precision = tp.divide(tp.add(fp).max(1));
var recall    = tp.divide(tp.add(fn).max(1));
var f1        = precision.multiply(recall).multiply(2)
                  .divide(precision.add(recall).max(0.001));

// Apply regression model to current glacier pixels only
var currentGlacierMask = glacier_2024.eq(1).and(peripheralMask);
var susceptibilityMap  = predictors
  .updateMask(currentGlacierMask)
  .classify(rfPredict)
  .rename('susceptibility')
  .max(0).min(1);

Precomputed Asset Architecture

Because Landsat compositing, distance raster computation, and machine learning inference are all computationally expensive at Greenland scale, all analytical outputs are exported once as a multi-band Earth Engine Image Asset. This approach decouples the heavy computation from the interactive application, allowing the app to load pre-calculated results at startup rather than recomputing them on every visit.

The exported asset stores the following bands, along with embedded model performance metadata (accuracy, kappa, F1, variable importance):

Band Content
glacier_2000 Binary glacier mask ~2000
glacier_2024 Binary glacier mask ~2024
retreat Cleaned retreat pixels
warming_rate LST trend (°C/yr, 2001–2024)
lst_early Mean summer LST ~2000
lst_recent Mean summer LST ~2024
susceptibility RF retreat susceptibility (0–1)
elevation ArcticDEM elevation (m)

Interactive Regional Statistics

The application supports click-based regional analysis to make susceptibility outputs actionable for end users. For any selected location, the system computes statistics within a 25 km radius buffer:

  • Glacier area change (km², ~2000 vs ~2024)
  • Surface temperature change (~2000 vs ~2024)
  • Decadal warming trend (°C/decade)
  • Mean elevation (m)
  • Mean retreat susceptibility score

This capability transforms the model from a static prediction surface into an exploratory decision-support tool, enabling policymakers and researchers to interrogate local conditions at any point across Greenland’s peripheral glacier zone.

Interface

The interface uses a linked split map. The left map shows the approximate 2000 glacier extent and early-period temperature context. The right map shows the approximate 2024 glacier extent, observed retreat, warming rate, susceptibility prediction, mining sites, and settlements. A swipe divider allows direct visual comparison between baseline and recent glacier conditions.

The side panel provides:

  • model performance statistics from the exported asset;
  • a variable-importance chart;
  • susceptibility and domain legends;
  • a click-based statistics panel.

When users click the map, the app summarises a 25 km radius region around the click location. It reports glacier area change, mean temperature, warming trend, elevation, domain status, and mean retreat susceptibility.

How it Works

1. Data Preparation and Precomputations

The app is organised as a three-script workflow. The first script, gee_preprocess_export.js, creates the remote-sensing and machine-learning layers described in the Methodology section. The second script, gee_context_export.js, exports lightweight context masks that separate the modelled peripheral glacier zone from the central/northern ice-sheet context. The final script, gee_glacier_mining_app.js, loads these exported assets and builds the user interface.

This separation is important because Greenland-wide Landsat compositing, distance rasters, and Random Forest inference are too computationally expensive to run whenever a user opens the app. Instead, the app reads a precomputed image asset:

var precomputed = ee.Image(
  'projects/ee-k24081637/assets/0025/greenland_glacier_mining_precomputed'
);

var contextPrecomputed = ee.Image(
  'projects/ee-k24081637/assets/0025/greenland_display_context_precomputed'
);

var glacier_2000 = precomputed.select('glacier_2000');
var glacier_2024 = precomputed.select('glacier_2024');
var retreatOnly  = precomputed.select('retreat').selfMask();
var warmingRate  = precomputed.select('warming_rate');
var susceptibilityMap = precomputed.select('susceptibility');

The key glacier-change calculation behind the asset is based on NDSI:

\[ NDSI = \frac{Green - SWIR1}{Green + SWIR1} \]

Retreat is defined as:

\[ R = 1 \quad \text{if} \quad I_{2000}=1 \ \text{and} \ I_{2024}=0 \]

where (I) is the binary glacier mask. Model performance is summarised using accuracy, kappa, precision, recall and F1:

\[ F1 = 2 \times \frac{Precision \times Recall}{Precision + Recall} \]

3. Click-Based Regional Statistics

To make the map actionable, users can click any location to generate statistics within a 25 km buffer. The app reports glacier area change, mean temperature, warming trend, elevation, domain status and mean susceptibility.

function handleMapClick(coords) {
  var point  = ee.Geometry.Point([coords.lon, coords.lat]);
  var region = point.buffer(25000);
  var pxArea = ee.Image.pixelArea();

  var stats = ee.Dictionary({
    area2000: glacier_2000.multiply(pxArea).reduceRegion({
      reducer: ee.Reducer.sum(),
      geometry: region,
      scale: clickStatsScale,
      maxPixels: 1e9
    }).get('glacier_2000'),

    area2024: glacier_2024.multiply(pxArea).reduceRegion({
      reducer: ee.Reducer.sum(),
      geometry: region,
      scale: clickStatsScale,
      maxPixels: 1e9
    }).get('glacier_2024'),

    susceptibility: susceptibilityMap.reduceRegion({
      reducer: ee.Reducer.mean(),
      geometry: region,
      scale: clickStatsScale,
      maxPixels: 1e9
    }).get('susceptibility')
  });
}

The app also checks whether the selected location falls inside the modelled peripheral glacier zone or only in the ice-sheet context layer. This prevents users from over-interpreting susceptibility outside the intended model domain.

var inModelDomain = s.domain !== null && s.domain > 0;

4. Limitations and Potential Expansion

The main limitations are data-related. Landsat NDSI may confuse seasonal snow, clouds, shadows and glacier ice; Landsat 7 baseline imagery may contain SLC-off gaps; and very high northern Greenland has weaker routine Landsat coverage. ArcticDEM is a mosaic rather than a single-date surface, so elevation and slope do not exactly match every Landsat year. GIMP is a static 2000 ice/ocean mask, while mining licences and drillholes indicate legal or exploration activity rather than confirmed active production. Settlement points are simplified accessibility indicators, and the 300 m modelling scale may smooth narrow outlet glaciers.

The susceptibility model therefore identifies spatial association, not causation. Mining proximity should be interpreted as an exploratory pressure indicator. Future work could compare a natural-only model with a full model including mining and settlement proximity:

var naturalBands = [
  'elevation', 'slope', 'warming_rate', 'lst_mean', 'coast_proximity'
];

var fullBands = [
  'elevation', 'slope', 'warming_rate', 'lst_mean',
  'coast_proximity', 'mine_proximity', 'settlement_proximity'
];
