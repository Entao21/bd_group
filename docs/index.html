Project Summary

This project develops an interactive Google Earth Engine application to explore glacier retreat susceptibility across Greenland’s peripheral glacier zone. This app is a combination of Landsat-derived glacier change, Landsat land surface temperature trends, ArcticDEM terrain variables, and proximity indicators for mining activity, settlements, and coastlines. A Random Forest model is then trained on observed retreat and stable-ice labels to map where current glaciers are more susceptible to retreat.

The application is aiming for providing a screening tool for identifying places where the ice sheet suffer the biggest risk of melting. This supports more transparent discussion around Arctic resource development, environmental monitoring, and glacier vulnerability.

Problem Statement

Greenland’s ice sheet is losing mass rapidly, with an average loss of about 150 ± 13 Gt per year between 1992 and 2020 (IMBIE Team, 2020). That amout of ice loss resulted in Greenland ice loss alone contributed to 14.7% of global sea-level rise per year.(IMBIE Team, 2020; Otosaka et al., 2023; van den Broeke et al., 2016). Climate change, global warming, human activities, mineral exploration, and many other factors are all threatening this sensitive cryospheric environment (Meredith et al.,2024).

Among all parts of Greenland’s ice sheet, peripheral glaciers are the most vulnerable ones. According to Larocca et al., 2023, peripheral glaciers are not only melting at a faster rate than the central ice sheet, but their melting speed is also accelerating more rapidly than the central. Research by Khan et al., 2022found that 4% of the peripheral ice sheet contributed about 14% of total Greenland ice loss (Khan et al., 2022).

Existing tools exhibits huge gap of showing retreat susceptibility. Our model is aiming for combining change detection, climate trends, topographic controls and human intervention indicators into a single exploratory geospatial tool that helps users investigate where these factors align with that gap by increased glacier retreat susceptibility.

End User

The application is designed for users who need a clear spatial overview rather than a static scientific map. Potential users include Arctic environmental researchers, Greenland planning and resource governance agencies, environmental NGOs, mining impact analysts, and students studying remote sensing or climate risk.

The interface supports these users by combining technical model outputs with accessible interaction: users can compare historical and recent glacier extent, toggle model layers, inspect model diagnostics, and click anywhere on the map to retrieve local glacier, temperature, elevation, and susceptibility statistics.

Methodology

The pipeline runs in three steps. Raw inputs — ArcticDEM terrain, Landsat 7/8/9 imagery, vector assets, and the Greenland AOI — feed four processing streams that produce a seven-variable predictor stack. A Random Forest model is trained on observed retreat and stable-ice labels and exports the results as an 8-band GEE image asset. A second script adds a 4-band context asset covering ice mask and domain boundaries. The interactive app then loads both assets alongside uploaded vector layers to build the split-panel interface — no heavy computation at runtime.

Glacier extent for two time windows — ~2000 (1999–2002) and ~2024 (2022–2024) — was mapped from Landsat summer composites (June–August). Each scene was cloud-masked using the QA_PIXEL band, and the Normalized Difference Snow Index (NDSI) was calculated from the green and SWIR bands (Bands 2 & 5 for Landsat 7; Bands 3 & 6 for Landsat 8/9). Pixels with NDSI > 0.4 were classified as glacier ice. Retreat is defined as any pixel that was ice in ~2000 but not in ~2024. Small isolated patches (fewer than 3 connected pixels) were removed from both retreat and stable-ice labels to reduce noise from cloud shadows and Landsat 7 scan-line gaps. All output rasters were standardized to a 300 m polar grid (EPSG:3413).

// Cloud masking using QA_PIXEL for Landsat 8/9
function maskL8(image) {
  var qa = image.select('QA_PIXEL');
  return image.updateMask(
    qa.bitwiseAnd(1 << 3).eq(0)          // cloud
      .and(qa.bitwiseAnd(1 << 4).eq(0))  // cloud shadow
  );
}

// NDSI from green and SWIR bands (Landsat 8/9)
function ndsiL8(img) {
  return img.addBands(
    img.normalizedDifference(['SR_B3', 'SR_B6']).rename('NDSI'));
}

// Build median NDSI composite and threshold at 0.4
function getGlacierExtent(startYear, endYear) {
  var t0 = ee.Date.fromYMD(startYear, 1, 1);
  var t1 = ee.Date.fromYMD(endYear, 12, 31);
  var summer = ee.Filter.calendarRange(6, 8, 'month');

  var l8 = ee.ImageCollection('LANDSAT/LC08/C02/T1_L2')
    .filterBounds(greenlandBox).filterDate(t0, t1)
    .filter(summer).map(maskL8).map(ndsiL8);
  var l9 = ee.ImageCollection('LANDSAT/LC09/C02/T1_L2')
    .filterBounds(greenlandBox).filterDate(t0, t1)
    .filter(summer).map(maskL8).map(ndsiL8);

  return l8.merge(l9).select('NDSI').median()
    .gt(0.4).rename('ice').clip(AOI_geom).unmask(0);
}

var glacier_2000 = getGlacierExtent(1999, 2002).rename('glacier_2000');
var glacier_2024 = getGlacierExtent(2022, 2024).rename('glacier_2024');

// Retreat = ice in 2000 but not in 2024
var retreatCandidate = glacier_2000.subtract(glacier_2024).max(0).gt(0);
var stableCandidate  = glacier_2000.eq(1).and(glacier_2024.eq(1));

// Remove isolated patches smaller than minPatchPixels (set to 3)
var cleanRetreat = retreatCandidate.selfMask()
  .connectedPixelCount(8, true).gte(minPatchPixels)
  .unmask(0).toByte().rename('retreat');

var cleanStable = stableCandidate.selfMask()
  .connectedPixelCount(8, true).gte(minPatchPixels)
  .unmask(0).toByte().rename('stable');

To track how temperatures have changed, Landsat surface temperature (ST) scenes from 2001 to 2024 were converted from raw DN values to degrees Celsius, then averaged into one composite per summer (June–August). Fitting a straight line through all 24 annual means gives a warming rate (°C/year) for every pixel. Two snapshot composites — one for ~2000 and one for ~2024 — were also retained so users can compare absolute temperatures directly in the app. Elevation and slope were derived from ArcticDEM v4 at the same 300 m resolution. Together with three proximity rasters, these variables make up the full predictor set:

// Convert Landsat ST_B10 DN to Celsius
function lstL8(image) {
  return image.select('ST_B10')
    .multiply(0.00341802).add(149.0).subtract(273.15)
    .rename('LST_C')
    .copyProperties(image, ['system:time_start']);
}

// Build annual summer LST composites for 2001-2024
var years = ee.List.sequence(2001, 2024);

var annualLST = ee.ImageCollection(years.map(function(year) {
  year = ee.Number(year);
  var s = ee.Date.fromYMD(year, 6, 1);
  var e = ee.Date.fromYMD(year, 8, 31);
  var l8 = ee.ImageCollection('LANDSAT/LC08/C02/T1_L2')
    .filterBounds(greenlandBox).filterDate(s, e).map(maskL8).map(lstL8);
  var l9 = ee.ImageCollection('LANDSAT/LC09/C02/T1_L2')
    .filterBounds(greenlandBox).filterDate(s, e).map(maskL8).map(lstL8);
  return l8.merge(l9).mean()
    .set('year', year).set('system:time_start', s.millis());
}));

// Attach numeric time band then run OLS — slope = °C/yr
var withTime = annualLST.map(function(img) {
  return img.addBands(
    ee.Image.constant(ee.Number(img.get('year'))).float().rename('time'));
});

var warmingRate = withTime.select(['time', 'LST_C'])
  .reduce(ee.Reducer.linearFit())
  .select('scale').clip(AOI_geom).rename('warming_rate');

// Proximity rasters: inverse cumulative cost distance, scaled 0-1
function distanceToFC(fc, maxDist) {
  var source = ee.Image(0).byte().paint(
    fc.map(function(f) { return f.buffer(1000); }), 1);
  return ee.Image(1).cumulativeCost(source, maxDist).unmask(maxDist);
}

var mineProximity = ee.Image(1)
  .subtract(distanceToFC(allMines, 100000).divide(100000))
  .max(0).min(1).rename('mine_proximity');

var settlementProximity = ee.Image(1)
  .subtract(distanceToFC(allSettlements, 100000).divide(100000))
  .max(0).min(1).rename('settlement_proximity');

var coastProximity = ee.Image(1)
  .subtract(distToCoast.divide(300000))
  .max(0).min(1).rename('coast_proximity');

A Random Forest with 100 trees was trained to score how likely each current glacier pixel is to retreat. Only pixels with clear historical retreat or clear stable-ice history were used as training examples — ambiguous pixels were left out. The model only runs on peripheral glaciers (GIMP land ice below 2500 m, plus all ice south of 64°N where the southern ice cap behaves like a peripheral glacier rather than the interior ice sheet).

To avoid the problem where nearby pixels leak from training into the test set, a spatial block validation split the map into 10 geographic chunks: chunks 0–6 trained the model and chunks 7–9 tested it. A second, classification-mode version of the same model produced the accuracy, kappa, precision, recall, and F1 scores shown in the app panel. The final score (0–1) is the fraction of the 100 trees that voted for retreat, assigned to every glacier pixel visible in 2022–2024.

// Build spatial folds from a lon/lat grid (10 folds total)
var lonLat = ee.Image.pixelLonLat();
var spatialFold = lonLat.select('longitude').add(180).divide(2).floor()
  .add(lonLat.select('latitude').add(90).divide(2).floor().multiply(13))
  .mod(10).rename('spatial_fold');

// Only sample clean retreat / stable pixels inside the model domain
var sampleMask = cleanRetreat.eq(1).or(cleanStable.eq(1)).and(peripheralMask);

var samples = predictors.addBands(target).addBands(spatialFold)
  .updateMask(sampleMask)
  .stratifiedSample({
    numPoints: 2000, classBand: 'retreat',
    region: greenlandBox, scale: exportScale, seed: 42,
    geometries: false, tileScale: 4
  });

var trainSet = samples.filter(ee.Filter.lt('spatial_fold', 7));
var testSet  = samples.filter(ee.Filter.gte('spatial_fold', 7));

// Classification RF → confusion matrix and performance stats
var rfClassify = ee.Classifier.smileRandomForest({numberOfTrees: 100, seed: 42})
  .train({ features: trainSet, classProperty: 'retreat',
           inputProperties: bandNames });

// Regression RF → continuous susceptibility score
var rfPredict = ee.Classifier.smileRandomForest({numberOfTrees: 100, seed: 42})
  .setOutputMode('REGRESSION')
  .train({ features: trainSet, classProperty: 'retreat',
           inputProperties: bandNames });

// Evaluate on held-out spatial blocks
var confusionMatrix = testSet.classify(rfClassify)
  .errorMatrix('retreat', 'classification', [0, 1]);
var precision = tp.divide(tp.add(fp).max(1));
var recall    = tp.divide(tp.add(fn).max(1));
var f1 = precision.multiply(recall).multiply(2)
           .divide(precision.add(recall).max(0.001));

// Score only current glacier pixels
var susceptibilityMap = predictors
  .updateMask(glacier_2024.eq(1).and(peripheralMask))
  .classify(rfPredict)
  .rename('susceptibility').max(0).min(1);

Precomputed Asset Architecture

Running Landsat composites, distance rasters, and the Random Forest across all of Greenland takes too long to repeat every time someone opens the app. Instead, everything is computed once and saved as a single multi-band Earth Engine image asset that the app reads at startup. Model performance metrics are embedded as asset metadata so the app can display them without re-running any calculations.

Interface

The interface uses a linked split map. The left map shows the approximate 2000 glacier extent and early-period temperature context. The right map shows the approximate 2024 glacier extent, observed retreat, warming rate, susceptibility prediction, mining sites, and settlements. A swipe divider allows direct visual comparison between baseline and recent glacier conditions.

The side panel provides:

  • model performance statistics from the exported asset;
  • a variable-importance chart;
  • susceptibility and domain legends;
  • a click-based statistics panel.

When users click the map, the app summarises a 25 km radius region around the click location. It reports glacier area change, mean temperature, warming trend, elevation, domain status, and mean retreat susceptibility.

Key Features Beyond the Application

While our application showcases the app’s core functionality, several critical features work behind the scenes to enhance its performance and reliability. In this section, we highlight key capabilities implemented to optimize efficiency and ensure a seamless user experience.

Multi-band Precomputation

To ensure a zero-latency user experience, the app avoids on-the-fly Landsat compositing by loading pre-processed 300m multi-band assets, containing glacier masks, LST trends, and RF susceptibility.

// Precomputing heavy layers into a single multi-band Asset
var precomputed = ee.Image.cat([
  glacier_2000.toByte(),
  glacier_2024.toByte(),
  warmingRate.toFloat(),
  susceptibilityMap.toFloat()
]).set({
  accuracy: confusionMatrix.accuracy(),
  f1: f1,
  importance: rfClassify.explain().get('importance')
});

Linked Dual-Map Interface

Utilizing a split-panel UI with a Map Linker, the app allows users to synchronize navigation between the year 2000 and 2024 to visually inspect glacier retreat alongside warming trends.

// Synchronizing two maps for historical comparison
var leftMap = ui.Map();
var rightMap = ui.Map();
var linker = ui.Map.Linker([leftMap, rightMap]);
var splitPanel = ui.SplitPanel({
  firstPanel: leftMap,
  secondPanel: rightMap,
  wipe: true,
  style: {stretch: 'both'}
});

Dynamic Regional Analysis

When a user clicks the map, the app dynamically generates a 25km buffer and performs on-the-fly spatial reductions to calculate localized glacier loss and temperature trends.

// Handling map clicks for regional statistics
function handleMapClick(coords) {
  var point = ee.Geometry.Point([coords.lon, coords.lat]);
  var region = point.buffer(25000); // 25km radius
  var stats = precomputed.reduceRegion({
    reducer: ee.Reducer.mean().combine(ee.Reducer.sum(), '', true),
    geometry: region,
    scale: 300
  });
}

Automated Workflow Funnel

The app features a 'Stage Funnel' dashboard that uses coarse 5km reductions at startup to provide an immediate nationwide summary of ice loss and risk tiers without hanging the UI.

// Fast dashboard summary using multi-scale reduction
var areaStats = areaBands.reduceRegion({
  reducer: ee.Reducer.sum(),
  geometry: AOI_geom,
  scale: 5000, // Coarser scale for responsive dashboard
  bestEffort: true
});

Model Transparency & Metrics

Instead of a 'black box', the app extracts RF model metadata (Accuracy, Kappa, Variable Importance) directly from asset properties and renders them using interactive Google Charts.

How it Works

1. Data Preparation and Precomputations

The app is organised as a three-script workflow. The first script, gee_preprocess_export.js, creates the remote-sensing and machine-learning layers described in the Methodology section. The second script, gee_context_export.js, exports lightweight context masks that separate the modelled peripheral glacier zone from the central/northern ice-sheet context. The final script, gee_glacier_mining_app.js, loads these exported assets and builds the user interface.

This separation is important because Greenland-wide Landsat compositing, distance rasters, and Random Forest inference are too computationally expensive to run whenever a user opens the app. Instead, the app reads a precomputed image asset:

var precomputed = ee.Image(
  'projects/ee-k24081637/assets/0025/greenland_glacier_mining_precomputed'
);

var contextPrecomputed = ee.Image(
  'projects/ee-k24081637/assets/0025/greenland_display_context_precomputed'
);

var glacier_2000 = precomputed.select('glacier_2000');
var glacier_2024 = precomputed.select('glacier_2024');
var retreatOnly  = precomputed.select('retreat').selfMask();
var warmingRate  = precomputed.select('warming_rate');
var susceptibilityMap = precomputed.select('susceptibility');

The processing steps behind these assets — NDSI glacier mapping, LST trend fitting, proximity rasters, and Random Forest training — are described in full in the Methodology section above. Model performance metrics (accuracy, kappa, precision, recall, F1) are stored as asset metadata and displayed in the app side panel without re-running any calculations.

3. Click-Based Regional Statistics

To make the map actionable, users can click any location to generate statistics within a 25 km buffer. The app reports glacier area change, mean temperature, warming trend, elevation, domain status and mean susceptibility.

function handleMapClick(coords) {
  var point  = ee.Geometry.Point([coords.lon, coords.lat]);
  var region = point.buffer(25000);
  var pxArea = ee.Image.pixelArea();

  var stats = ee.Dictionary({
    area2000: glacier_2000.multiply(pxArea).reduceRegion({
      reducer: ee.Reducer.sum(),
      geometry: region,
      scale: clickStatsScale,
      maxPixels: 1e9
    }).get('glacier_2000'),

    area2024: glacier_2024.multiply(pxArea).reduceRegion({
      reducer: ee.Reducer.sum(),
      geometry: region,
      scale: clickStatsScale,
      maxPixels: 1e9
    }).get('glacier_2024'),

    susceptibility: susceptibilityMap.reduceRegion({
      reducer: ee.Reducer.mean(),
      geometry: region,
      scale: clickStatsScale,
      maxPixels: 1e9
    }).get('susceptibility')
  });
}

The app also checks whether the selected location falls inside the modelled peripheral glacier zone or only in the ice-sheet context layer. This prevents users from over-interpreting susceptibility outside the intended model domain.

var inModelDomain = s.domain !== null && s.domain > 0;

4. Limitations and Potential Expansion

The main limitations are data-related. Landsat NDSI may confuse seasonal snow, clouds, shadows and glacier ice; Landsat 7 baseline imagery may contain SLC-off gaps; and very high northern Greenland has weaker routine Landsat coverage. ArcticDEM is a mosaic rather than a single-date surface, so elevation and slope do not exactly match every Landsat year. GIMP is a static 2000 ice/ocean mask, while mining licences and drillholes indicate legal or exploration activity rather than confirmed active production. Settlement points are simplified accessibility indicators, and the 300 m modelling scale may smooth narrow outlet glaciers.

The susceptibility model therefore identifies spatial association, not causation. Mining proximity should be interpreted as an exploratory pressure indicator. Future work could compare a natural-only model with a full model including mining and settlement proximity:

var naturalBands = [
  'elevation', 'slope', 'warming_rate', 'lst_mean', 'coast_proximity'
];

var fullBands = [
  'elevation', 'slope', 'warming_rate', 'lst_mean',
  'coast_proximity', 'mine_proximity', 'settlement_proximity'
];
