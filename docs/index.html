Project Summary

This project develops an interactive Google Earth Engine application to explore glacier retreat susceptibility across Greenland’s peripheral glacier zone. The app combines Landsat-derived glacier change, Landsat land surface temperature trends, ArcticDEM terrain variables, and proximity indicators for mining activity, settlements, and coastlines. A Random Forest model is trained on observed retreat and stable-ice labels to map where current glaciers are more susceptible to retreat.

The purpose is not to prove that mining caused glacier retreat. Instead, the application provides a screening tool for identifying places where observed retreat, climate warming, terrain conditions, and mining-related proximity overlap. This supports more transparent discussion around Arctic resource development, environmental monitoring, and glacier vulnerability.

Problem Statement

Greenland’s peripheral glaciers are changing rapidly under Arctic warming, while renewed mineral exploration creates planning questions in sensitive cryospheric environments. Existing glacier products and mining datasets are usually viewed separately, making it difficult for non-specialist users to compare climate-driven change with human activity indicators.

This project addresses that gap by building a web-based decision-support tool that:

  • maps glacier extent change from approximately 2000 to 2024;
  • estimates summer land surface temperature trends;
  • integrates elevation, slope, mining proximity, settlement proximity, and coastal proximity;
  • predicts retreat susceptibility using a validated Random Forest model;
  • communicates model outputs through an interactive split-map interface.

End User

The application is designed for users who need a clear spatial overview rather than a static scientific map. Potential users include Arctic environmental researchers, Greenland planning and resource governance agencies, environmental NGOs, mining impact analysts, and students studying remote sensing or climate risk.

The interface supports these users by combining technical model outputs with accessible interaction: users can compare historical and recent glacier extent, toggle model layers, inspect model diagnostics, and click anywhere on the map to retrieve local glacier, temperature, elevation, and susceptibility statistics.

Methodology

The methodology follows a three-script workflow. Heavy raster processing and model training are run first, display-context masks are exported second, and the final application only loads precomputed assets. This architecture makes the app faster and avoids repeated large computations at start-up.

Glacier Change Detection

Glacier extent is estimated from Landsat summer composites using the Normalized Difference Snow Index (NDSI). NDSI separates snow and ice from darker land surfaces because snow has high green reflectance and low shortwave-infrared reflectance.

\[ NDSI = \frac{Green - SWIR1}{Green + SWIR1} \]

Landsat 7 uses bands SR_B2 and SR_B5, while Landsat 8/9 use SR_B3 and SR_B6. Pixels with NDSI > 0.4 are treated as glacier/snow-ice pixels. Two multi-year windows are compared: 1999-2002 as the baseline and 2022-2024 as the recent period. The multi-year windows reduce the influence of single-year cloud, snow, or sensor noise.

var glacier_2000 = getGlacierExtent(1999, 2002);
var glacier_2024 = getGlacierExtent(2022, 2024);

var rawRetreat = glacier_2000.subtract(glacier_2024).max(0);
var retreatCandidate = rawRetreat.gt(0);
var stableCandidate = glacier_2000.eq(1).and(glacier_2024.eq(1));

The model labels are deliberately simple but cleaned: “retreat” means a pixel was mapped as glacier in the baseline window and not in the recent window; “stable” means it remained glacier in both windows. Small isolated patches are removed using connected-pixel filtering, reducing salt-and-pepper noise before sampling.

Predictor Engineering

Seven predictor variables are sampled to a common modelling scale. This does not mean all source datasets have the same native resolution. Landsat and ArcticDEM have different original resolutions, but Earth Engine samples them consistently at the modelling scale during training and export.

The predictor stack includes:

  • elevation from ArcticDEM;
  • slope derived from ArcticDEM;
  • Landsat summer LST warming rate from 2001-2024;
  • early-period mean LST;
  • proximity to mining licences and drillholes;
  • proximity to settlements and cities;
  • proximity to the coastline.
var predictors = ee.Image.cat([
  elevation.rename('elevation'),
  slope.rename('slope'),
  warmingRate.rename('warming_rate'),
  lstEarly.rename('lst_mean'),
  mineProximity.rename('mine_proximity'),
  settlementProximity.rename('settlement_proximity'),
  coastProximity.rename('coast_proximity')
]);

Mining is represented as a proximity variable rather than a direct impact radius. This is more defensible: it lets the model test whether distance to known mining activity helps explain retreat susceptibility after climate and terrain variables are included, without claiming a causal mining effect.

Random Forest Susceptibility Model

The Random Forest model is trained to distinguish cleaned retreat pixels from stable glacier pixels. Sampling is stratified so both classes are represented, and a spatial block split is used for validation so nearby pixels are less likely to appear in both the training and testing sets.

var samples = predictors.addBands(target).stratifiedSample({
  numPoints: 2000,
  classBand: 'retreat',
  region: AOI_geom,
  scale: 300,
  seed: 42,
  geometries: true,
  tileScale: 4
});

var trainSet = samples.filter(ee.Filter.lt('fold', 7));
var testSet  = samples.filter(ee.Filter.gte('fold', 7));

var rf = ee.Classifier.smileRandomForest({
  numberOfTrees: 100,
  seed: 42
}).train({
  features: trainSet,
  classProperty: 'retreat',
  inputProperties: bandNames
});

Model performance is reported using confusion matrix outputs, accuracy, kappa, precision, recall, F1 score, and variable importance. The app reads these diagnostics from asset metadata, so users can see the model quality without rerunning the classifier.

Model Domain

The susceptibility model is focused on Greenland’s peripheral glacier zone rather than the high-elevation central ice sheet. This is important because peripheral glaciers are more directly comparable to the small coastal glaciers and ice margins where mining access, settlements, and terrain gradients are meaningful. The central ice sheet is still displayed as context, but the app avoids presenting it as a directly modelled susceptibility zone.

var modelDomain = gimpIceMask
  .updateMask(elevation.lt(2500).or(latitude.lt(64)))
  .rename('model_domain');

var iceSheetContext = gimpIceMask
  .updateMask(elevation.gte(2500).and(latitude.gte(64)))
  .rename('ice_sheet_context');

var northIceContext = gimpIceMask
  .updateMask(latitude.gt(82.6))
  .rename('north_ice_context');

The northern context layer is included because routine Landsat WRS-2 coverage is limited at very high latitudes. Showing this area as context helps communicate coverage uncertainty instead of silently ignoring it.

Precomputed Asset Architecture

The final app does not recompute Landsat composites, distance rasters, or Random Forest predictions. Instead, two export scripts create reusable Earth Engine image assets:

  • gee_preprocess_export.js exports the analytical model bands, including glacier extent, retreat labels, warming rate, early/recent LST, elevation, and susceptibility.
  • gee_context_export.js exports lightweight display masks for the modelled peripheral zone, central ice-sheet context, and northern context.
  • gee_glacier_mining_app 2.js loads those assets and builds the interface.
Export.image.toAsset({
  image: exportImage,
  description: 'greenland_glacier_mining_precomputed',
  assetId: outputAsset,
  region: greenlandBox,
  crs: 'EPSG:3413',
  scale: 300,
  maxPixels: 1e13
});

This improves app reliability because users interact with already-computed layers rather than triggering large server-side operations every time the web app opens.

Interface

The interface uses a linked split map. The left map shows the approximate 2000 glacier extent and early-period temperature context. The right map shows the approximate 2024 glacier extent, observed retreat, warming rate, susceptibility prediction, mining sites, and settlements. A swipe divider allows direct visual comparison between baseline and recent glacier conditions.

The side panel provides:

  • model performance statistics from the exported asset;
  • a variable-importance chart;
  • susceptibility and domain legends;
  • a click-based statistics panel.

When users click the map, the app summarises a 25 km radius region around the click location. It reports glacier area change, mean temperature, warming trend, elevation, domain status, and mean retreat susceptibility.

var region = point.buffer(25000);

How it Works

1. Data Preparation and Precomputations

The app uses a precompute-first workflow. gee_preprocess_export.js runs the expensive Landsat, terrain, proximity, and Random Forest steps once. gee_context_export.js exports lightweight model-domain masks. gee_glacier_mining_app 2.js then loads these assets for interactive display. This avoids recomputing Greenland-wide raster analysis every time the app opens.

var countries    = ee.FeatureCollection('USDOS/LSIB_SIMPLE/2017');
var AOI          = countries.filter(ee.Filter.eq('country_na', 'Greenland'));
var AOI_geom     = AOI.geometry();
var greenlandBox = ee.Geometry.Rectangle([-73, 59, -12, 84]);

var exportCrs   = 'EPSG:3413';
var exportScale = 300;
var minPatchPixels = 3;

Glacier extent is mapped from summer Landsat NDSI, using a baseline window (1999-2002) and a recent window (2022-2024). The NDSI formula is:

\[ NDSI = \frac{Green - SWIR1}{Green + SWIR1} \]

Pixels with NDSI > 0.4 are treated as glacier/snow-ice pixels. Multi-year summer composites reduce the influence of cloud, seasonal snow, and single-image noise.

function ndsiL7(image) {
  return image.normalizedDifference(['SR_B2', 'SR_B5']).rename('NDSI');
}

function ndsiL8(image) {
  return image.normalizedDifference(['SR_B3', 'SR_B6']).rename('NDSI');
}

function getGlacierExtent(startYear, endYear) {
  var t0 = ee.Date.fromYMD(startYear, 1, 1);
  var t1 = ee.Date.fromYMD(endYear, 12, 31);
  var summer = ee.Filter.calendarRange(6, 8, 'month');
  var ndsiComposite;

  if (endYear <= 2012) {
    ndsiComposite = ee.ImageCollection('LANDSAT/LE07/C02/T1_L2')
      .filterBounds(greenlandBox)
      .filterDate(t0, t1)
      .filter(summer)
      .map(maskL7)
      .map(ndsiL7)
      .select('NDSI')
      .median();
  } else {
    var l8 = ee.ImageCollection('LANDSAT/LC08/C02/T1_L2')
      .filterBounds(greenlandBox)
      .filterDate(t0, t1)
      .filter(summer)
      .map(maskL8)
      .map(ndsiL8);

    var l9 = ee.ImageCollection('LANDSAT/LC09/C02/T1_L2')
      .filterBounds(greenlandBox)
      .filterDate(t0, t1)
      .filter(summer)
      .map(maskL8)
      .map(ndsiL8);

    ndsiComposite = l8.merge(l9).select('NDSI').median();
  }

  return ndsiComposite.gt(0.4);
}

var glacier_2000 = getGlacierExtent(1999, 2002).rename('glacier_2000');
var glacier_2024 = getGlacierExtent(2022, 2024).rename('glacier_2024');

var rawRetreat = glacier_2000.subtract(glacier_2024).max(0);
var retreatCandidate = rawRetreat.gt(0);
var stableCandidate = glacier_2000.eq(1).and(glacier_2024.eq(1));

var cleanRetreat = retreatCandidate
  .updateMask(retreatCandidate)
  .connectedPixelCount(8, true)
  .gte(minPatchPixels)
  .rename('retreat');

var cleanStable = stableCandidate
  .updateMask(stableCandidate)
  .connectedPixelCount(8, true)
  .gte(minPatchPixels)
  .rename('stable');

The retreat label is:

\[ R = 1 \quad \text{if} \quad I_{2000}=1 \ \text{and} \ I_{2024}=0 \]

where (I) is the binary glacier mask. Stable ice is defined where (I_{2000}=1) and (I_{2024}=1). Connected-pixel filtering removes isolated noisy labels.

Temperature trend is calculated from Landsat Surface Temperature. Each summer year is converted to Celsius, then a per-pixel linear fit estimates warming rate from 2001-2024.

function lstL8(image) {
  return image.select('ST_B10')
    .multiply(0.00341802)
    .add(149.0)
    .subtract(273.15)
    .rename('LST_C');
}

var annualLST = ee.ImageCollection(
  ee.List.sequence(2001, 2024).map(function(y) {
    y = ee.Number(y);
    var summerMean = getAnnualSummerLST(y);
    return summerMean.addBands(ee.Image.constant(y).rename('time')).float();
  })
);

var warmingRate = annualLST
  .select(['time', 'LST_C'])
  .reduce(ee.Reducer.linearFit())
  .select('scale')
  .rename('warming_rate');

The predictor stack combines terrain, climate, and proximity variables. Mining is represented as proximity, not as a direct causal effect:

\[ P = 1 - \frac{\min(d, d_{max})}{d_{max}} \]

where (d) is distance to the nearest mining feature. This lets the model test whether mining proximity is associated with retreat susceptibility after terrain and warming are included.

var predictors = ee.Image.cat([
  elevation.rename('elevation'),
  slope.rename('slope'),
  warmingRate.rename('warming_rate'),
  lstEarly.rename('lst_mean'),
  mineProximity.rename('mine_proximity'),
  settlementProximity.rename('settlement_proximity'),
  coastProximity.rename('coast_proximity')
]);

var bandNames = [
  'elevation', 'slope', 'warming_rate', 'lst_mean',
  'mine_proximity', 'settlement_proximity', 'coast_proximity'
];

The Random Forest model is trained with stratified retreat/stable samples. A spatial block split is used instead of a purely random split, reducing leakage between nearby pixels.

var trainingImage = predictors
  .addBands(cleanRetreat.rename('retreat'))
  .addBands(spatialFold)
  .updateMask(sampleMask);

var samples = trainingImage.stratifiedSample({
  numPoints: 2000,
  classBand: 'retreat',
  region: greenlandBox,
  scale: exportScale,
  seed: 42,
  geometries: false,
  tileScale: 4
});

var trainSet = samples.filter(ee.Filter.lt('spatial_fold', 7));
var testSet  = samples.filter(ee.Filter.gte('spatial_fold', 7));

var rfClassify = ee.Classifier.smileRandomForest({
  numberOfTrees: 100,
  seed: 42
}).train({
  features: trainSet,
  classProperty: 'retreat',
  inputProperties: bandNames
});

The validation set is used to calculate accuracy, kappa, precision, recall, F1 score, and variable importance. F1 is included because glacier retreat labels can be imbalanced:

\[ F1 = 2 \times \frac{Precision \times Recall}{Precision + Recall} \]

var validated = testSet.classify(rfClassify);
var confusion = validated.errorMatrix('retreat', 'classification');
var cmArray = ee.Array(confusion.array());

var TP = cmArray.get([1, 1]);
var FP = cmArray.get([0, 1]);
var FN = cmArray.get([1, 0]);

var precision = TP.divide(TP.add(FP));
var recall = TP.divide(TP.add(FN));
var f1 = precision.multiply(recall).multiply(2)
  .divide(precision.add(recall));

var importance = ee.Dictionary(rfClassify.explain().get('importance'));

The preprocessing output is exported as a multi-band image asset. Model statistics are stored as metadata for the app panel.

var precomputed = ee.Image.cat([
  glacier_2000.toByte(),
  glacier_2024.toByte(),
  cleanRetreat.toByte(),
  warmingRate.float(),
  lstEarly.float(),
  lstRecent.float(),
  susceptibilityMap.float(),
  elevation.float()
]).set({
  accuracy: confusion.accuracy(),
  kappa: confusion.kappa(),
  precision: precision,
  recall: recall,
  f1: f1,
  n_train: trainSet.size(),
  n_test: testSet.size(),
  export_scale_m: exportScale,
  validation_split: 'spatial block split: folds 0-6 train, 7-9 test'
});

Export.image.toAsset({
  image: precomputed,
  description: 'greenland_glacier_mining_precomputed',
  assetId: outputAsset,
  region: greenlandBox,
  crs: exportCrs,
  scale: exportScale,
  maxPixels: 1e13
});

The context export separates the modelled peripheral glacier zone from central/high-elevation ice-sheet context. This helps users see Greenland as a whole while avoiding over-interpretation outside the intended model domain.

var gimpIceMask = ee.Image('OSU/GIMP/2000_ICE_OCEAN_MASK')
  .select('ice_mask')
  .eq(2)
  .rename('gimp_ice_mask');

var latitude = ee.Image.pixelLonLat().select('latitude');

var modelDomain = gimpIceMask
  .and(elevation.lt(2500).or(latitude.lt(64)))
  .rename('model_domain');

var iceSheetContext = gimpIceMask
  .and(modelDomain.not())
  .rename('ice_sheet_context');

var northIceContext = gimpIceMask
  .and(latitude.gte(78))
  .rename('north_ice_context');

3. Insights and Objectives

The app supports exploratory decision-making around glacier change and mining activity. It provides four main outputs: observed glacier change, Random Forest retreat susceptibility, variable importance, and click-based local statistics. The output is a screening map for further monitoring, not proof that mining caused glacier retreat.

var a0  = (s.area2000 / 1e6).toFixed(1);
var a1  = (s.area2024 / 1e6).toFixed(1);
var dA  = (a1 - a0).toFixed(1);
var pct = a0 > 0 ? ((dA / a0) * 100).toFixed(1) : 'N/A';

clickStatsPanel.add(ui.Label(
  '~2000: ' + a0 + ' km2   ~2024: ' + a1 + ' km2'
));

clickStatsPanel.add(ui.Label(
  'Change: ' + dA + ' km2 (' + pct + '%)'
));

The continuous susceptibility value is converted into a simple label for communication.

var rPct = (s.susceptibility * 100).toFixed(0);
var rLabel = s.susceptibility > 0.7 ? 'HIGH' :
             s.susceptibility > 0.4 ? 'MODERATE' : 'LOW';

clickStatsPanel.add(ui.Label(
  rLabel + ' (' + rPct + '%)',
  {fontWeight: 'bold'}
));

4. Limitations and Potential Expansion

The main limitations come from the datasets and modelling design. Landsat NDSI can confuse seasonal snow, cloud, shadow, and glacier ice, and Landsat 7 baseline imagery may contain scan-line gaps. Very high northern Greenland has weaker routine Landsat coverage, so northern context is treated cautiously. ArcticDEM is a high-resolution mosaic but not a single-date surface, so elevation and slope do not exactly match every Landsat year. GIMP is a static 2000 ice/ocean mask and cannot represent all later ice-front changes. Mining licences and drillholes indicate legal/exploration activity, not necessarily active production, and the licence dataset is a recent snapshot compared with the 2000-2024 glacier period. Settlement points are simplified accessibility indicators rather than detailed population exposure. Finally, 300 m resampling improves feasibility but smooths narrow outlet glaciers.

The model learns spatial association, not causation. Mining proximity should therefore be read as an exploratory pressure indicator. Updating the app requires rerunning the two export scripts.

// Workflow limitation:
// 1. Run gee_preprocess_export.js
// 2. Wait for greenland_glacier_mining_precomputed to finish
// 3. Run gee_context_export.js
// 4. Wait for greenland_display_context_precomputed to finish
// 5. Run or publish gee_glacier_mining_app 2.js

Future work could compare a natural-only model with a full model including mining and settlement proximity. If the full model performs better under spatial validation, mining proximity would have stronger explanatory value, while still avoiding causal overclaiming.
